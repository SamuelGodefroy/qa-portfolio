# Naming Convention for projects

To keep our test documentation structured, we use the following format for Test Case IDs:

**Format**: `[PROJECT]-[TICKET_TYPE]-[MODULE]-[NUMBER]`

*   **[PROJECT]**: Project code name (e.g., `AS` for AgentStack).
*   **[TICKET_TYPE]**: Сode name for the type of work performed:
   
| Code | Definition |
| :--- | :--- |
| `TC` | *Test Case* |
| `BUG` | *Bug-report* |
|`ISS` | *Issue in the project repository* |

*   **[MODULE_TYPE]**: Short module identifier (e.g., `AUTH` for Authentication, `UI_LANGUAGE` for Language Interface, `UI_FRONT` for Interface frontend).
   
*   **[NUMBER]**: Document identification number (e.g., `001`, `002`).

**Example**:

`AS-TC-AUTH-001` -> AgentStack, Test Case, Authentication Module, Test Case 001.
